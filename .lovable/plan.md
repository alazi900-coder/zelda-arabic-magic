

## تشخيص المشكلة

من سجلات الأخطاء، المشاكل الرئيسية هي:

1. **Gemini API quota exhausted (429)**: مفتاح Gemini المجاني استنفد الحصة اليومية (`gemini-2.0-flash-lite` free tier limit = 0). النظام ينتقل تلقائياً إلى Lovable AI.

2. **فشل تحليل JSON من Lovable AI**: الترجمات الطويلة (خاصة الحوارات) تُنتج JSON مبتور أو يحتوي على أقواس/علامات تنصيص غير محمية داخل النصوص العربية، مما يسبب خطأ 500.

3. **حجم الدفعة كبير (10 نصوص)**: عند إرسال نصوص حوارية طويلة معاً، يتجاوز الرد حد الـ tokens فيُبتر.

## الخطة

### 1. ترقية نموذج Gemini المباشر
- تغيير من `gemini-2.0-flash-lite` إلى `gemini-2.0-flash` (حصة مجانية أكبر بكثير)
- في حالة الـ fallback إلى Lovable AI، النموذج الحالي `google/gemini-2.5-flash` جيد

### 2. إضافة آلية إعادة المحاولة (Retry)
- عند فشل JSON parsing، إعادة المحاولة مرة واحدة بدفعة أصغر (تقسيم الدفعة إلى نصفين)
- عند خطأ 429، انتظار المدة المحددة في `retryDelay` ثم إعادة المحاولة

### 3. تحسين متانة تحليل JSON
- إضافة استراتيجية إصلاح إضافية: استخراج كل مفتاح `K{n}` بشكل فردي عبر regex بدل الاعتماد على `JSON.parse` فقط
- معالجة الحالات التي يحتوي فيها النص العربي على علامات تنصيص مزدوجة غير محمية

### 4. تقليل حجم الدفعة الديناميكي
- تقليل `AI_BATCH_SIZE` من 10 إلى 5 للنصوص الطويلة (الحوارات)
- أو تطبيق حد ديناميكي بناءً على إجمالي حروف الدفعة

### 5. تحسين الـ Prompt
- إضافة تعليمات صريحة لتجنب علامات التنصيص المزدوجة داخل الترجمة
- تعليمات أوضح للحفاظ على تنسيق JSON صالح

## التفاصيل التقنية

### الملفات المتأثرة:
- **`supabase/functions/translate-entries/index.ts`**: ترقية النموذج، إضافة retry، تحسين JSON parser
- **`src/components/editor/types.tsx`**: تقليل `AI_BATCH_SIZE` أو جعله ديناميكياً

### تحسين `extractJsonObject`:
```text
1. محاولة JSON.parse العادية
2. إصلاح trailing commas + control chars
3. (جديد) استخراج regex لكل K{n}: /"K(\d+)"\s*:\s*"((?:[^"\\]|\\.)*)"/g
4. (جديد) إذا فشل كل شيء، تقسيم الدفعة وإعادة المحاولة
```

translateWithAI() {
